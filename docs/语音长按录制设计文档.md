# 语音输入长按录制设计文档

## 1. 需求概述

将当前语音输入（WKVoiceInputView）的交互方式从"点击开始 → 点击停止"改为"长按录音 → 松手发送"，参考豆包App的交互体验。

## 2. 交互状态定义

### 2.1 三种状态

| 状态 | 触发条件 | 视觉表现 |
|------|---------|---------|
| **Idle（闲置）** | 默认 / 松手后 | 显示麦克风按钮，文字"按住 说话" |
| **Recording（录音中）** | 长按按钮不松手 | 全屏蓝色渐变蒙层 + 波形 + "松手发送，上移取消" |
| **Cancelling（取消模式）** | 录音中手指上移超过阈值 | 蒙层变为红色渐变 + 红色波形 + "松手取消" |

> Transcribing（转写中）状态保持不变，松手后自动进入。

### 2.2 手势流程

```
手指按下麦克风按钮
    │
    ├─ 150ms内松手 → 忽略（防误触）
    │
    ├─ 超过150ms → 开始录音，显示蓝色蒙层
    │     │
    │     ├─ 手指保持不动 / 在按钮区域内
    │     │    → 状态：Recording
    │     │    → 文字："松手发送，上移取消"
    │     │    → 蒙层：蓝色渐变
    │     │
    │     ├─ 手指上移超过120pt
    │     │    → 状态：Cancelling
    │     │    → 文字："松手取消"
    │     │    → 蒙层：红色渐变
    │     │    → 波形变红
    │     │
    │     ├─ 手指从上方回到按钮区域
    │     │    → 恢复 Recording 状态
    │     │
    │     └─ 手指松开
    │          ├─ Recording状态 → 停止录音 → 转写
    │          └─ Cancelling状态 → 取消录音 → 回到Idle
    │
    └─ 录音时长 < 1秒松手 → 提示"说话时间太短"→ 回到Idle
```

## 3. 视觉设计

### 3.1 蒙层（Overlay）

录音开始后，在聊天页面上方覆盖一层半透明渐变蒙层。

**Recording 蓝色蒙层：**
```
┌─────────────────────────┐
│    聊天消息（模糊透出）    │
│                         │
│                         │
│   "松手发送，上移取消"    │  ← 灰白色文字，居中偏上
│                         │
│  ●●●●●●●●●●●●●●●●●●●●  │  ← 白色波形条，底部居中
│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│  ← 蓝色渐变（从上到下加深）
└─────────────────────────┘
```

- 渐变方向：从上到下
- 顶部：`rgba(173, 216, 255, 0.3)` (浅蓝半透明)
- 底部：`rgba(59, 130, 246, 0.85)` (深蓝)
- 蒙层高度：覆盖输入面板以上的区域（约屏幕下半部分）

**Cancelling 红色蒙层：**
- 顶部：`rgba(255, 200, 200, 0.3)` (浅红半透明)
- 底部：`rgba(239, 68, 68, 0.75)` (红色)

### 3.2 提示文字

| 状态 | 文字内容 | 颜色 | 字号 |
|------|---------|------|------|
| Recording | "松手发送，上移取消" | `rgba(255,255,255,0.7)` | 15pt Medium |
| Cancelling | "松手取消" | `rgba(255,100,100,1.0)` | 15pt Medium |

位置：蒙层垂直居中偏上

### 3.3 波形条

- 数量：约40-50个小圆点/短柱（参考效果图密集排列）
- Recording：白色，`rgba(255,255,255,0.9)`
- Cancelling：红色，`rgba(239,68,68,0.8)`
- 位置：蒙层底部，距底边约60pt
- 高度随音量动态变化（2pt ~ 20pt）
- 更新频率：100ms

## 4. 修改范围

### 4.1 主要修改文件

| 文件 | 修改内容 |
|------|---------|
| `WKVoiceInputView.h` | 新增 `WKVoiceInputStateCancelling` 状态 |
| `WKVoiceInputView.m` | 核心改造：手势替换、蒙层UI、状态管理 |

### 4.2 状态枚举变更

```objc
typedef NS_ENUM(NSInteger, WKVoiceInputState) {
    WKVoiceInputStateIdle,          // 闲置
    WKVoiceInputStateRecording,     // 录音中（松手发送）
    WKVoiceInputStateCancelling,    // 取消模式（松手取消）
    WKVoiceInputStateTranscribing,  // 转写中
};
```

### 4.3 手势改造

**移除：**
- `onMicTapped`（点击事件）

**新增：**
- `UILongPressGestureRecognizer`，绑定到麦克风按钮
- `minimumPressDuration = 0.15`（150ms防误触）
- 在手势回调中处理 `Began / Changed / Ended / Cancelled` 四个阶段

```
Began   → startRecording + 显示蒙层
Changed → 计算手指Y偏移，判断是否进入取消区域
Ended   → Recording状态则转写，Cancelling状态则取消
Cancelled → 取消录音
```

### 4.4 取消区域判定

```
手指按下位置 touchStartY
当前手指位置 currentY
偏移量 = touchStartY - currentY（上移为正）

if (偏移量 > 120pt) → Cancelling
else               → Recording
```

带回弹区间：进入取消后需要回到 80pt 以内才恢复为 Recording（避免边界抖动）。

### 4.5 蒙层视图结构

```
recordingOverlay (UIView, 全屏)
  ├─ gradientLayer (CAGradientLayer, 渐变背景)
  ├─ hintLabel (UILabel, "松手发送，上移取消")
  ├─ waveformView (UIView, 波形容器)
  │    └─ waveBars[] (UIView[], 40-50个柱形条)
  └─ 不拦截蒙层以外的触摸（userInteractionEnabled根据需要设置）
```

蒙层添加到 `WKVoiceInputView` 的 superview（即 WKConversationInputPanel 或更上层），确保覆盖聊天区域。

### 4.6 动画

**蒙层出现：**
- 从透明到目标渐变，0.2秒 ease-in

**蒙层消失：**
- Recording松手：0.15秒 fade out，同时切换到 Transcribing
- Cancelling松手：0.2秒 fade out，回到 Idle

**Recording ↔ Cancelling 切换：**
- 渐变色 0.25秒渐变过渡（蓝→红 / 红→蓝）
- 波形颜色同步 0.25秒过渡
- 文字 crossfade 0.2秒

## 5. 保持不变的部分

| 功能 | 说明 |
|------|------|
| 转写流程 | Thinking进度条动画、API调用、结果回调完全不变 |
| 录音参数 | 16kHz AAC 单声道，不变 |
| 权限处理 | 首次使用时请求麦克风权限，不变 |
| 最大时长 | 默认60秒，到时自动停止并转写，不变 |
| chat_context | 聊天上下文收集逻辑不变 |
| 代理协议 | WKVoiceInputViewDelegate 接口不变 |
| CWVoiceView | Tab容器管理不变 |
| 辅助按钮 | @/空格/删除/换行按钮在Idle状态保持不变 |

## 6. 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 长按后快速松手（< 1秒） | 提示"说话时间太短"，蒙层 fade out，回到 Idle |
| 录音中App进入后台 | 自动取消录音，蒙层消失 |
| 录音中来电 | 音频中断通知，自动取消 |
| 录音达到最大时长 | 自动停止 → 转写（与当前逻辑一致） |
| 手指滑出屏幕 | 等同于松手，按当前状态处理 |
| 麦克风权限被拒 | 长按后弹出权限设置引导，不进入录音 |
| 转写中再次长按 | 忽略（Transcribing状态不响应手势） |
